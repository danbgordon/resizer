# Resizer

## Project Summary

A native macOS Swift app that resizes any application's window to exact pixel dimensions. Works with any app, and offers viewport-accurate sizing for supported browsers (Chrome, Safari, Edge, Arc, Brave, Chromium). Sizes are loaded from a user-editable config file.

Built as a safe, zero-dependency alternative to Chrome extensions (which have a history of malware issues ‚Äî e.g., the original Resizer extension).

Target audience: Fleet (fleetdm.com) team members who need to record their screen at specific dimensions.

## What Exists

- **`resizer.swift`** ‚Äî Single-file Swift app. All logic: config loading, app detection, Accessibility API window management, browser viewport measurement, UI (NSAlert).
- **`build.sh`** ‚Äî Developer build script. Compiles Swift, packages `.app` bundle, codesigns, generates `.pkg` installer. Resets permission flags on rebuild.
- **`README.md`** ‚Äî User-facing setup guide covering install, usage, custom sizes, permissions, deployment, and troubleshooting.

## Key Technical Details

- Native Swift app compiled with `swiftc`, packaged as `.app` bundle (ad-hoc codesigned)
- Window resize uses **Accessibility API** (`AXUIElement`) ‚Äî only requires Accessibility permission (one time, for all apps)
- No per-app Automation permission needed for window resizing (unlike the old AppleScript approach)
- Auto-detects the previously-active app using `lsappinfo visibleProcessList` (skips Finder, Dock, Spotlight, Launchpad)
- Sizes loaded from `~/.config/resizer/sizes.conf` (CSV format: `width,height`), falls back to built-in defaults
- Config file created on first launch if missing
- Viewport resize (browsers only): two-pass technique ‚Äî set window size, measure viewport via JavaScript (`osascript` subprocess), adjust to compensate for browser UI chrome
  - Chromium browsers: `using terms from application "Google Chrome"` + `execute javascript`
  - Safari: `do JavaScript` in front document
  - Requires Automation permission per browser (only for viewport mode, with user-facing explanation)
- Multi-monitor aware: keeps window on its current screen via `NSScreen` bounds clamping
- `AXIsProcessTrusted()` may cache in-process; bundled `ax_check` binary used for polling during Accessibility grant flow
- `LSUIElement=true` in Info.plist ‚Äî no persistent Dock icon while running
- Bundle ID: `com.danbgordon.resizer`
- `.pkg` installer generated by `build.sh` for MDM deployment (installs to `/Applications`)
